## 지금 상태에서 되는 것

이 페이지는 완성된 책 본문이 아니라, 앞으로의 교재를 올릴 수 있는 최소 문서 파이프라인입니다.

- MDX 기반 본문 렌더링
- Fumadocs sidebar / TOC / search
- Code Hike 기반 커스텀 코드 블록
- diff annotation 렌더링 준비

## 왜 이 준비가 먼저 필요할까

책을 실제로 읽히게 만들려면, 글만 있는 정적 문서보다 훨씬 많은 것이 필요합니다.

먼저는 아래가 안정적으로 연결되어야 합니다.

1. 챕터 원고를 MDX로 쓸 수 있어야 한다
2. 코드 블록을 단순 문자열이 아니라 UI 컴포넌트처럼 다룰 수 있어야 한다
3. 나중에 퀴즈와 진행률 같은 인터랙션을 얹을 수 있어야 한다

> 이 앱은 그 출발점을 만드는 단계입니다.

## Code Hike 샘플

아래 코드는 나중에 책에서 보여줄 "작은 diff 워크스루"의 가장 단순한 예시입니다.

```ts index.ts
export function installProject() {
  const phase = "resolve";

  // !diff -
  return `${phase}: read package.json`;

  // !diff +
  return `${phase}: read package.json and prepare install plan`;
}
```

이제부터 할 일은 이 컴포넌트 위에 다음을 얹는 것입니다.

- 챕터별 코드 스텝 전환
- line focus
- scrollycoding
- 퀴즈 블록
- 진도 저장
