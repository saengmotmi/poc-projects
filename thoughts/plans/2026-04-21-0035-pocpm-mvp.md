# Plan: pocpm MVP

Date: 2026-04-21 00:35 KST
Input: `thoughts/research/2026-04-21-0035-pocpm-mvp.md`

## Goal

Add an internal package `@poc/pocpm` that demonstrates the minimal package-manager loop with TDD:

- resolve exact npm versions and `workspace:*`
- fetch tarballs into a local store
- link a hoist-less `node_modules`
- create `.bin`
- run root scripts through installed binaries

## Non-goals

- Do not replace Yarn as the repo package manager in this change.
- Do not implement semver solving, peer deps, hoisting, or publish flows.
- Do not support Windows-specific link shims in this MVP.

## Risks and Mitigations

- Risk: tests become flaky if they hit the real npm registry.
  - Mitigation: use a local mock registry server with deterministic tarballs.
- Risk: recursive install logic becomes difficult to inspect.
  - Mitigation: prefer fixture-driven integration tests over broad unit mocks.
- Risk: workspace linking may leak stale `node_modules`.
  - Mitigation: clear install targets before each install pass.

## Definition of Done

- `packages/pocpm` exists with build, check, and test scripts.
- Failing tests are added first for install and run behavior.
- Tests cover exact npm resolution, transitive install, workspace symlink linking, `.bin`, lockfile, and cache.
- Implementation satisfies all tests.
- Root `build/check/test` pass under Node `20.19.0`.

## Validation Commands

- `fnm exec --using 20.19.0 corepack yarn workspace @poc/pocpm test`
- `fnm exec --using 20.19.0 corepack yarn workspace @poc/pocpm check`
- `fnm exec --using 20.19.0 corepack yarn build`
- `fnm exec --using 20.19.0 corepack yarn test`
